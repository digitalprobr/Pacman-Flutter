# Pacman Flutter — Version 2 Planning

## Current Architecture (v1 Snapshot)

| Concern | Implementation |
|---|---|
| State management | Single `_HomePageState` with raw `setState` |
| Map / Grid | Flat `List<int> barriers` hardcoded in the widget state |
| Ghost AI | 3 separate copy-pasted `moveGhostN()` methods, direction-lock walk |
| Game loop | 3 independent `Timer.periodic` calls inside `startGame()` |
| Food | `List<int> food` filled by iterating all non-barrier cells |
| Player input | `GestureDetector` swipe — no on-screen D-pad |
| Audio | 4 `AudioPlayer` instances, file names hardcoded |
| Scoring | In-memory `int score` — lost on close |
| Lives | None |
| Levels / Phases | None — single infinite grid, same map forever |
| Power-ups | None |
| Navigation | Single screen (`main.dart` → `HomePage`) |
| Tests | Only the auto-generated `widget_test.dart` |

---

## Features for Version 2

### 1. Multiple Phases / Levels

**Why:** The biggest missing piece — the game never advances.

- Define a `LevelConfig` model with:
  - `List<int> barriers` — the walls for that level
  - `Duration ghostSpeed` — gets faster each level
  - `int ghostCount` — level 3+ adds a 4th ghost
  - `String bgColor` — visual variety between phases
- Start with 3 hand-crafted maps (classic, tight corridors, open arena).
- Trigger level advance when `food.isEmpty`.
- Show a short "Level N!" overlay between phases instead of restarting from scratch.
- Carry `score` and `lives` across levels.

---

### 2. Lives System

- Add `int lives = 3` to game state.
- On ghost collision: decrement lives, reset positions, do NOT reset score or food.
- Show a lives HUD (3 small Pacman icons in the bottom bar).
- Only show "Game Over" dialog when `lives == 0`.

---

### 3. Power Pellets (Energizers)

- Place 4 special cells per map (corners) as `powerPellets`.
- On collection: enter `frightened` mode for ~7 seconds.
  - Ghosts render as the classic blue-flashing sprite.
  - Ghost collision in this mode scores points (200 → 400 → 800 → 1600) and respawns the ghost at its start position.
- Play `pacman_eatghost.wav` on ghost kill.

---

### 4. Smarter Ghost AI

Current AI is direction-lock (wall-bounce), with no chasing. Upgrade each ghost to a distinct personality:

| Ghost | Behavior |
|---|---|
| Blinky (red) | Directly targets Pacman's current cell (BFS / Manhattan distance) |
| Pinky (pink) | Targets 4 cells ahead of Pacman's facing direction |
| Inky (blue) | Keeps patrolling a fixed quadrant; switches to scatter on fear |
| Clyde (orange) | Chases when far, scatters when within 8 cells |

- Extract a shared `Ghost` model class to remove the 3 copy-pasted `moveGhostN()` methods.
- Implement a simple BFS pathfinder that is called every N ticks (not every frame) to avoid performance issues on mobile.

---

### 5. Architecture Refactor (State Management)

The current 900-line `HomePage.dart` mixing UI, game logic, timers, and audio is unmaintainable.

Proposed split:

```
lib/
  main.dart
  app/
    routes.dart
  models/
    level_config.dart       ← map definition, barriers, pellets
    ghost_model.dart        ← position, personality, state enum
    game_state.dart         ← score, lives, level, phase enum
  controllers/
    game_controller.dart    ← all Timer logic, movement, collision
    ghost_controller.dart   ← AI per ghost
    audio_controller.dart   ← single AudioController service
  screens/
    home_screen.dart        ← start / high-score menu
    game_screen.dart        ← grid + HUD (read-only, no logic)
    game_over_screen.dart
  widgets/
    grid_cell.dart
    hud_bar.dart
    lives_indicator.dart
    ghost_widget.dart       ← takes a color/sprite param, replaces ghost.dart × 3
    power_pellet_widget.dart
```

Use **`provider`** or **`riverpod`** for `GameState` to avoid threading `setState` calls through timers.

---

### 6. On-Screen D-Pad Controls

- Add an optional D-pad overlay in the bottom-right corner for players who prefer tap over swipe.
- Keep swipe support as the primary input.
- Store preference in `SharedPreferences`.

---

### 7. High Score & Persistence

- Use `shared_preferences` to persist:
  - Top-5 high scores with initials
  - Last completed level
- Show a high-score screen accessible from the start menu.

---

### 8. Bonus Items (Fruit)

- Spawn a fruit widget at the center of the map after 70 and 170 pellets are eaten.
- Award bonus points: cherry (100), strawberry (300), orange (500).
- Fruit disappears after 9 seconds if not collected.

---

### 9. Visual & UX Polish

- Add a **Start Menu** screen with animated Pacman logo, Play, High Scores, and Settings.
- Animate level transition with a brief flash and "READY!" countdown.
- Make the grid **responsive** — currently `numberInRow = 11` is hardcoded; compute it from `MediaQuery` width so it looks correct on tablets and foldables.
- Replace `GridView.builder` with a `CustomPainter` for faster rendering and precise pixel-art styling.

---

### 10. Sound Improvements

- Consolidate all 4 `AudioPlayer` instances into a single `AudioController` service.
- Add missing classic sounds: `eatghost.wav`, `extrapac.wav` (extra life at 10 000 pts).
- Respect system silent/vibrate mode using `audio_session` package.

---

### 11. Testing

- Unit tests for `GameController` (collision, scoring, level advance).
- Unit tests for `GhostController` (AI pathfinding, frightened state).
- Widget tests for `HudBar` lives display.
- Integration test: play through level 1 autonomously using simulated input.

## Known Bugs to Fix First

- `moveGhost3()` reads `ghostLast` (ghost 1's direction) instead of `ghostLast3` — so ghost 3 mirrors ghost 1's movement.
- Timers started inside `startGame()` are never cancelled: pressing PLAY a second time spawns duplicate timer loops causing double-speed movement and audio overlap.
- `preGame` is set to `false` before food is fully initialized, causing the grid to briefly show empty cells on first render.
- `audioPaused` is re-instantiated every call to `startGame()` but the old instance is never disposed.
